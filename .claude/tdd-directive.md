【TDD自動オーケストレーション指示 / 自動注入】

このプロジェクトは TDD を標準フローとする（詳細は `claude.md §6`、エージェント定義は `.claude/agents/`）。
コードを書く作業（機能追加・仕様変更・バグ修正・リファクタ）では、メインの応答者は必ず
`tdd-orchestrator` として振る舞い、Agent ツールで各サブエージェントへ委譲すること。

ルーティング判断:
- 質問・調査・説明のみ／設定変更／ドキュメントだけの依頼 → TDDフローは不要。通常対応でよい。
- 実装コードに手を入れる依頼 → 下記の TDD サイクルを必ず適用する。

TDDサイクル（1サイクル = 1振る舞い、`claude.md §6.1`）:
1. 分解: 要件をテスト可能な振る舞いに分解し、1つ選ぶ。
2. RED: `test-author` サブエージェント（Agentツール, subagent_type/name: test-author）に
   失敗するテストの作成を委譲。`npm run test -- <path>` で赤と失敗理由（実装不足）を確認。
3. GREEN: テストを通す最小限の実装を書く（過剰実装しない）。
4. REFACTOR: 緑を保って整理。
5. 検証: `coverage-guard` サブエージェントに `npm run test` / `test:coverage` /
   `typecheck` / `lint` の一括実行とカバレッジ70%判定を委譲。
6. 必要に応じ `ts-safety` / `schema-authz-guard` / `code-reviewer` へ委譲。

厳守事項:
- 失敗するテストを書く前に実装コードを書き始めない。
- カバレッジしきい値(70%)を下げる・対象除外する等で緑を偽装しない。
- 不変条件（140文字・owner/ADMINS権限・CMS更新権限・招待コード検証）はテストで担保する。
